## 项目概述

**jij** 是一个用 Rust 编写的操作系统智能代理，将自然语言转换为可控的 Linux / macOS / Windows 系统操作。

**核心特性**：
- 自然语言交互：用中文描述任务，Agent 自动规划并执行
- 多平台支持：Linux / macOS / Windows 均可运行
- 零配置启动：自动检测已设置的 API Key，无需额外配置文件
- 工具驱动架构：所有操作通过工具调用完成，安全可控
- 五级风险评估：Safe → Low → Medium → High → Critical
- TUI 界面：基于 ratatui 的实时交互终端界面
- Playbook 模板：保存常用多步骤任务，一键执行
- 反向解释模式：读取配置文件并用自然语言解释
- Watchdog 监控：后台监控资源使用，异常告警
- SSH 远程模式：连接远程服务器执行操作

## 安装与配置

### 编译

**环境要求**：Rust 1.75+（推荐 1.80+）、任意平台

```bash
git clone https://github.com/jry21223/agent-unix.git
cd agent-unix

# Release 编译
cargo build --release

# 编译产物
./target/release/jij --version
# jij 0.2.1
```

### 预编译二进制

从 [GitHub Releases](https://github.com/jry21223/agent-unix/releases) 下载对应平台的二进制：

| 平台 | 文件名 |
|------|--------|
| macOS Apple Silicon | `jij-macos-arm64` |
| Linux x86_64 | `jij-linux-x86_64` |
| Windows x86_64 | `jij-windows-x86_64.exe` |

```bash
# macOS / Linux
chmod +x jij-macos-arm64
./jij-macos-arm64 chat

# Windows
jij-windows-x86_64.exe chat
```

### 添加到 PATH

**macOS / Linux**：

```bash
# 方式一：复制到 ~/.local/bin（推荐，无需 sudo）
mkdir -p ~/.local/bin
cp target/release/jij ~/.local/bin/jij

# 确认 PATH（~/.zshrc 或 ~/.bashrc）
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

# 方式二：全局安装（需要 sudo）
sudo cp target/release/jij /usr/local/bin/jij
```

**Windows**：将 `jij.exe` 所在目录添加到系统 `PATH` 环境变量。

**验证**：
```bash
jij --version   # jij 0.2.1
jij --help
```

### 配置 API Key

#### 方式一：环境变量（推荐，零配置）

**jij 启动时自动检测已设置的 API Key**，无需运行配置向导。

```bash
# Anthropic Claude（推荐，原生 tool_use，自动检测）
export ANTHROPIC_API_KEY=sk-ant-你的key

# OpenAI GPT（自动检测）
export OPENAI_API_KEY=sk-你的key

# DeepSeek（自动检测）
export DEEPSEEK_API_KEY=你的key

# Groq（自动检测）
export GROQ_API_KEY=你的key

# 智谱 GLM（自动检测）
export GLM_API_KEY=你的key

# Moonshot Kimi（自动检测）
export KIMI_API_KEY=你的key

# 阿里云百炼（自动检测）
export BAILIAN_API_KEY=你的key

# 通用 Key（最高优先级，覆盖其他变量）
export AGENT_UNIX_LLM_API_KEY=你的key
```

启动时 jij 会显示检测结果：
```
✅ 自动检测到 anthropic 配置，无需额外设置
```

**自动检测优先级**：`AGENT_UNIX_LLM_API_KEY` > `ANTHROPIC_API_KEY` > 各 Provider 专属 Key > `OPENAI_API_KEY`

#### 方式二：交互式配置向导

```bash
jij config --setup               # 交互式向导：选择 Provider、填写 Key
jij config --show                # 查看当前配置
jij config --list                # 列出所有支持的 Provider
jij config --delete              # 删除配置文件
```

配置文件保存路径：`~/.jij/config.json`（Windows：`%USERPROFILE%\.jij\config.json`）

#### 方式三：自定义端点

```bash
export AGENT_UNIX_LLM_PROVIDER=anthropic
export AGENT_UNIX_LLM_BASE_URL=https://your-proxy.example.com
export AGENT_UNIX_LLM_MODEL=claude-sonnet-4-5
export AGENT_UNIX_LLM_API_KEY=your-key
jij chat
```

## 快速上手

```bash
# 1. 设置 API Key（任选一种）
export ANTHROPIC_API_KEY=sk-ant-你的key

# 2. 进入交互式 TUI 对话
jij chat

# 3. 或执行单条指令
jij run "查看磁盘使用情况"

# 4. 预览模式（不实际执行）
jij run --dry-run "清理 30 天前的日志"

# 5. 查看内置 Playbook
jij playbooks

# 6. 启动资源监控
jij watch --duration 60
```

## 核心功能详解

### 交互式对话（TUI 模式）

```bash
jij chat              # 默认 TUI 界面（推荐）
jij chat --no-tui     # 传统 CLI 模式
jij chat --mode safe  # 安全模式（Medium 级也需确认）
```

启动后 Agent 自动采集系统环境（OS、CPU、内存、磁盘、服务），然后进入对话循环。

**对话内置指令**：

| 指令 | 说明 |
|------|------|
| `/help` | 显示帮助 |
| `/status` | 实时系统状态 |
| `/history` | 本次操作历史 |
| `/undo` | 撤销上一步操作 |
| `/clear` | 清除对话上下文 |
| `/report` | 系统健康综合报告 |
| `/playbook list` | 列出 Playbook |
| `/playbook save <名称>` | 保存当前操作为 Playbook |
| `/playbook run <名称>` | 执行 Playbook |
| `/voice tts` | 开启/关闭语音朗读 |
| `/exit` | 退出 |

**TUI 快捷键**：

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Y` | 复制最后一条 Agent 回复到剪贴板 |
| `Ctrl+P / N` | 浏览历史输入 |
| `PgUp / PgDn` | 滚动对话区 |
| `↑ / ↓` | 逐行滚动 |
| `End` | 滚到底部 |
| `Ctrl+C` / `Esc` | 退出 |

**示例会话**：

```
🤖 jij v0.2.1 已就绪

【系统环境】
  OS：macOS 15.4  主机：MacBook-Air.local
  CPU：10 cores, Apple M2  内存：16.0G total, 8.2G used
  磁盘：250G total, 70.1G free, 72% used

👤 你：查看磁盘使用情况

  🔧 Step 1: system.info({"query": "disk"})
     ✅ 磁盘总容量 250GB，已用 180GB，可用 70GB  (45ms)

🤖 Agent：磁盘使用率 72%，剩余 70GB 可用。
         建议检查 /var/log 和 ~/Downloads 释放空间。

👤 你：删除 /etc/passwd

🚨 操作被强制阻止（CRITICAL）
   原因：删除系统关键文件会导致系统无法正常运行
```

### 单条指令模式

```bash
jij run "查看磁盘使用情况"
jij run "内存占用最高的 5 个进程"
jij run "安装 nginx"                        # 根据 OS 自动选包管理器
jij run "把 nginx 配置到 8080 端口并重启"   # 多步骤自动拆解
jij run --dry-run "清理 /var/log 下 30 天前的日志"  # 预览不执行
jij run --mode safe "停止 nginx 服务"        # 安全模式
```

### Playbook 任务模板

```bash
# 查看 Playbook
jij playbooks

# 对话中执行
/playbook run system-health-check
/playbook save my-backup 5       # 保存最近 5 步为 my-backup
/playbook list nginx             # 搜索 nginx 相关
```

**Playbook 三级来源**（高优先级覆盖低优先级）：

| 来源 | 路径 |
|------|------|
| 内置 (Bundled) | 系统预定义 |
| 用户级 (User) | `~/.jij/playbooks/*.json` |
| 项目级 (Project) | `./.jij/playbooks/*.json` |

**自定义 Playbook 格式**（`~/.jij/playbooks/my-check.json`）：

```json
{
  "name": "my-disk-check",
  "description": "检查磁盘并显示详情",
  "steps": [
    { "tool": "system.info", "args": { "query": "disk" }, "reason": "检查使用率", "dry_run": false },
    { "tool": "shell.exec", "args": { "command": "df -h" }, "reason": "详细信息", "dry_run": false }
  ],
  "created_at": "2026-04-23T00:00:00Z",
  "run_count": 0
}
```

### 反向解释模式

```bash
jij explain                          # 列出支持的文件类型
jij explain /etc/nginx/nginx.conf    # 解释 nginx 配置
jij explain /etc/ssh/sshd_config     # 解释 SSH 配置
jij explain /etc/passwd              # 解释系统用户文件
```

### Watchdog 监控系统

```bash
jij watch --duration 120   # 后台监控 120 秒
```

默认告警规则：磁盘 > 80%（Warning）/ > 95%（Critical），内存 > 85% / > 95%。

在 TUI 对话模式中，Watchdog 会自动在后台运行，告警实时推送到对话区。

### SSH 远程模式

```bash
# 连接远程服务器
jij --ssh root@192.168.1.100 chat
jij --ssh admin@server.example.com:2222 --ssh-key ~/.ssh/id_ed25519 chat

# 远程单条指令
jij --ssh root@192.168.1.100 run "查看磁盘使用情况"
```

## 安全机制

所有工具调用在执行前经过五级风险评估：

| 级别 | 行为 | 示例 |
|------|------|------|
| **Safe** | 直接执行 | 查询磁盘、内存信息 |
| **Low** | 直接执行 | 查看文件内容 |
| **Medium** | `--mode safe` 下需确认 | 创建文件、安装软件 |
| **High** | 弹窗要求确认（TUI）/ 输入 `yes` | 停止服务、修改系统配置 |
| **Critical** | **直接拒绝，不可确认** | `rm -rf /`、删除 `/etc/passwd` |

```bash
jij --mode safe chat    # Medium 及以上都需确认
jij --mode normal chat  # 默认：High 及以上需确认
jij --mode auto chat    # 只拦截 Critical
```

## 完整命令参考

```
jij <COMMAND> [OPTIONS]

命令：
  chat       交互式对话（默认 TUI）
  run        执行单条自然语言指令
  config     查看/配置 LLM Provider
  history    查看操作历史
  undo       撤销上一次操作
  playbooks  列出 Playbook 模板
  explain    反向解释配置文件
  watch      启动后台监控
  info       显示系统信息
  help       显示帮助
```

**全局参数**：

| 参数 | 说明 | 默认 |
|------|------|------|
| `--verbose` | 详细日志 | 关闭 |
| `--mode` | `safe` / `normal` / `auto` | `normal` |
| `--provider` | `anthropic` / `openai` / `deepseek` / `groq` / `glm` / `kimi` / `bailian` / `minimax` / `openrouter` / `custom` | 自动检测 |
| `--model` | 模型 ID | Provider 默认 |
| `--base-url` | 自定义 API 端点 | Provider 默认 |
| `--no-tui` | 禁用 TUI | 关闭 |
| `--ssh` | SSH 目标（`user@host[:port]`） | 无 |
| `--ssh-key` | SSH 密钥文件路径 | `~/.ssh/id_rsa` |

## 常见问题

**Q1: 启动提示 `未找到 API Key`**

直接设置环境变量，无需配置文件：
```bash
export ANTHROPIC_API_KEY=sk-ant-xxx   # 或 OPENAI_API_KEY、DEEPSEEK_API_KEY 等
jij chat  # 自动检测并使用
```

**Q2: macOS 上 `systemctl: command not found`**

macOS 没有 systemctl，使用 `launchctl`。jij 会根据 OS 自动选择合适的命令。

**Q3: 如何切换到其他模型？**

```bash
# 使用 GLM（国内可直连）
export GLM_API_KEY=xxx
jij chat  # 自动检测 GLM

# 手动指定
jij --provider deepseek --model deepseek-chat chat
```

**Q4: undo 能撤销哪些操作？**

| 操作 | 回滚方案 |
|------|----------|
| `apt install nginx` | `apt remove nginx` |
| `useradd testuser` | `userdel testuser` |
| `systemctl start sshd` | `systemctl stop sshd` |
| `rm file.txt` | 无法自动回滚（建议用 dry-run 预览） |

**Q5: Windows 上如何使用？**

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-xxx"
.\jij.exe chat
```
所有功能均支持 Windows，系统信息使用 `wmic` 获取，剪贴板使用 `clip.exe`。

## 项目架构

```
src/
├── main.rs              # CLI 入口（clap）
├── config.rs            # LLM Provider 配置 + 环境变量自动检测
├── user_config.rs       # 配置向导 + 持久化
├── agent/
│   ├── loop.rs          # Agent 核心循环（ReAct）
│   ├── memory.rs        # 会话历史 + 智能压缩
│   └── planner.rs       # 任务分解（单步/多步/消歧）
├── llm/
│   ├── client.rs        # LLM API 调用（Anthropic / OpenAI-compatible）
│   └── prompt.rs        # System Prompt 构建（注入系统上下文）
├── tools/
│   ├── shell.rs         # shell.exec（跨平台：sh/cmd）
│   ├── file.rs          # file.read / file.write / file.search
│   └── system.rs        # system.info（disk/memory/cpu/process）
├── safety/
│   ├── classifier.rs    # 五级风险分类器
│   ├── patterns.rs      # CRITICAL / HIGH 规则库
│   └── audit.rs         # 审计日志（JSON Lines）
├── ui/
│   ├── app.rs           # TUI 主事件循环
│   ├── renderer.rs      # 帧渲染
│   ├── state.rs         # 全量渲染状态（含跨平台剪贴板）
│   └── widgets/         # chat / input / status / modal / statusbar
├── context/
│   └── system_scan.rs   # 跨平台系统信息采集
├── executor/
│   ├── local.rs         # 本地命令执行（跨平台）
│   └── ssh.rs           # SSH 远程执行
├── playbook/            # Playbook 多来源加载与管理
├── explainer/           # 反向解释模式
├── watchdog/            # 后台监控系统
├── voice/               # TTS 语音朗读
└── types/
    ├── tool.rs          # ToolCall / Playbook / RollbackPlan
    └── risk.rs          # RiskLevel 枚举
```
