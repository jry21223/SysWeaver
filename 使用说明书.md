## 项目概述

**Agent Unix** 是一个用 Rust 编写的操作系统智能代理，将自然语言转换为可控的 Linux / macOS 系统操作。

**核心特性**：
- 自然语言交互：用中文描述任务，Agent 自动执行
- 工具驱动架构：所有操作通过工具调用完成，安全可控
- 五级风险评估：Safe → Low → Medium → High → Critical
- Playbook 模板：保存常用多步骤任务，一键执行
- 反向解释模式：读取配置文件并用自然语言解释
- Watchdog 监控：后台监控资源使用，异常告警

## 安装与配置

### 编译

**环境要求**：Rust 1.75+（推荐 1.80+）、macOS / Linux

```bash
# 克隆或进入项目目录
cd /path/to/agent-unix

# Release 编译（约 40 秒，会下载依赖）
cargo build --release

# 编译产物路径
./target/release/agent-unix --version
# agent-unix 0.1.0
```

### 添加到 PATH

编译成功后，将二进制安装到系统 PATH，使 `agent-unix` 命令全局可用。

**方式一：复制到 `~/.local/bin`（推荐，无需 sudo）**

```bash
# 确认目录存在
mkdir -p ~/.local/bin

# 复制二进制
cp target/release/agent-unix ~/.local/bin/agent-unix
chmod +x ~/.local/bin/agent-unix

# 确认 ~/.local/bin 在 PATH 中（zsh）
echo $PATH | grep -q "$HOME/.local/bin" && echo "已在 PATH 中" || echo "需要添加"

# 如未在 PATH 中，添加到 ~/.zshrc（或 ~/.bashrc）
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**方式二：安装到 `/usr/local/bin`（需要 sudo，所有用户可用）**

```bash
sudo cp target/release/agent-unix /usr/local/bin/agent-unix
sudo chmod +x /usr/local/bin/agent-unix
```

**验证安装**：

```bash
which agent-unix       # 应输出路径
agent-unix --version   # agent-unix 0.1.0
agent-unix --help      # 查看完整帮助
```

### 配置 API Key

Agent Unix 支持多种 LLM Provider，通过环境变量或 CLI 参数配置。

**Anthropic Claude（推荐，原生 tool_use）**

```bash
export ANTHROPIC_API_KEY=sk-ant-你的key
```

**OpenAI**

```bash
export OPENAI_API_KEY=sk-你的key
agent-unix --provider openai chat
```

**OpenRouter（多模型聚合）**

```bash
export AGENT_UNIX_LLM_API_KEY=sk-or-你的key
agent-unix --provider openrouter --model openai/gpt-4o chat
```

**Groq（超快推理）**

```bash
export AGENT_UNIX_LLM_API_KEY=gsk_你的key
agent-unix --provider groq --model llama-3.1-70b-versatile chat
```

**DeepSeek**

```bash
export AGENT_UNIX_LLM_API_KEY=你的key
agent-unix --provider deepseek chat
```

**自定义端点（代理/中转）**

```bash
export AGENT_UNIX_LLM_PROVIDER=anthropic
export AGENT_UNIX_LLM_BASE_URL=https://your-proxy.example.com
export AGENT_UNIX_LLM_MODEL=claude-sonnet-4-5
export AGENT_UNIX_LLM_API_KEY=your-key
agent-unix chat
```

**环境变量优先级**：CLI 参数 > `AGENT_UNIX_LLM_*` 变量 > 官方变量（`ANTHROPIC_API_KEY` / `OPENAI_API_KEY`）

### 交互式配置向导

首次使用建议通过配置向导完成设置：

```bash
agent-unix config --setup    # 交互式向导：选择 Provider、输入 Key、选择模型
agent-unix config --show     # 查看当前配置
agent-unix config --list     # 列出所有支持的 Provider
agent-unix config --delete   # 删除配置文件
```

## 核心功能详解

### 交互式对话（TUI 模式）

```bash
agent-unix chat              # 默认 TUI 界面
agent-unix chat --no-tui     # 传统 CLI 模式（用于脚本/无交互终端）
agent-unix chat --mode safe  # 安全模式（Medium 级操作也需确认）
```

启动后 Agent 会先采集系统环境（OS、CPU、内存、磁盘、服务），然后进入对话循环。

**对话内置指令**：

| 指令 | 说明 |
|------|------|
| 自然语言任务 | `"查看磁盘使用情况"` |
| `history` | 查看本次会话操作历史 |
| `undo` | 撤销上一步操作（有回滚方案时生效） |
| `exit` / `quit` | 退出对话 |

**示例会话**：

```
🤖 Agent Unix v0.1.0
   正在采集系统环境...
   ✅ 系统：macOS 15.4 @ MacBook-Air.local

👤 你：查看磁盘使用情况

🔧 Step 1: system.info({"query": "disk"})
   ✅ 成功：磁盘总容量 250GB，已用 180GB，可用 70GB

🤖 Agent：磁盘使用率 72%，剩余 70GB。建议定期清理日志文件。

👤 你：列出内存占用最高的 5 个进程

🔧 Step 1: shell.exec({"command": "ps aux --sort=-%mem | head -6"})
   ✅ 成功

👤 你：删除 /etc/passwd

🚨 操作被强制阻止（CRITICAL）
   原因：删除系统关键文件可能导致系统无法启动

👤 你：停止 sshd 服务

⚠️ 高风险操作 — 需要您确认
   工具：shell.exec
   命令：sudo systemctl stop sshd
   风险：停止 SSH 服务后将无法远程登录
输入 'yes' 确认，其他键取消 › _
```

### 单条指令模式

适合在脚本中调用或快速查询：

```bash
# 基础查询
agent-unix run "查看磁盘使用情况"
agent-unix run "内存占用最高的 5 个进程"
agent-unix run "当前系统负载"

# 环境感知操作（根据 OS 自动选择包管理器）
agent-unix run "安装 nginx"

# 多步骤任务
agent-unix run "把 nginx 配置到 8080 端口并重启"

# 预览模式（不实际执行，仅显示计划）
agent-unix run --dry-run "清理 /var/log 下 30 天前的日志"
agent-unix run --dry-run "创建用户 testuser"

# 安全模式
agent-unix run --mode safe "停止 nginx 服务"

# 使用详细日志
agent-unix run --verbose "查看网络连接状态"
```

### Playbook 任务模板

Playbook 是预定义的多步骤任务，类似 Ansible Playbook，适合固定流程的批量操作。

#### 查看内置 Playbook

```bash
agent-unix playbooks
```

输出示例：
```
📋 已加载的 Playbook：
   来源统计：内置 3，用户 0，项目 0，覆盖 0

   • system-health-check — 检查系统健康状态：磁盘、内存、CPU、关键服务
     步骤数：3，运行次数：0

   • install-web-server — 安装并配置 Nginx Web 服务器
     步骤数：3，运行次数：0

   • cleanup-old-logs — 清理 30 天前的旧日志文件（安全模式）
     步骤数：3，运行次数：0
```

#### Playbook 三级来源

| 来源 | 路径 | 优先级 |
|------|------|--------|
| Bundled（内置） | 系统预定义 | 最低 |
| User（用户级） | `~/.agent-unix/playbooks/*.json` | 中 |
| Project（项目级） | `.agent-unix/playbooks/*.json` | 最高 |

同名 Playbook 高优先级覆盖低优先级。

#### 在对话中执行 Playbook

```bash
agent-unix chat

# 然后输入：
"运行 system-health-check playbook"
"执行系统健康检查模板"
"按照 cleanup-old-logs 清理日志"
```

#### 创建自定义 Playbook

```bash
mkdir -p ~/.agent-unix/playbooks
```

创建 `~/.agent-unix/playbooks/my-check.json`：

```json
{
  "name": "my-disk-check",
  "description": "检查磁盘并显示详情",
  "steps": [
    {
      "tool": "system.info",
      "args": { "query": "disk" },
      "reason": "检查磁盘使用率",
      "dry_run": false
    },
    {
      "tool": "shell.exec",
      "args": { "command": "df -h" },
      "reason": "详细磁盘信息",
      "dry_run": false
    }
  ],
  "created_at": "2026-04-22T00:00:00Z",
  "run_count": 0
}
```

验证加载：
```bash
agent-unix playbooks
# 应显示：内置 3，用户 1，项目 0
```

### 反向解释模式

读取系统配置文件，生成自然语言解释，适合不熟悉某配置项时快速理解。

```bash
# 列出支持的文件类型
agent-unix explain

# 解释指定文件
agent-unix explain /etc/passwd
agent-unix explain /etc/nginx/nginx.conf
agent-unix explain /etc/ssh/sshd_config
agent-unix explain /etc/fstab
```

**支持的文件类型**：

| 类型 | 示例路径 |
|------|----------|
| 配置文件 | `/etc/nginx/nginx.conf`、`/etc/ssh/sshd_config`、`/etc/crontab` |
| 日志文件 | `/var/log/syslog`、`/var/log/auth.log` |
| 系统文件 | `/etc/passwd`、`/etc/fstab`、`/etc/hostname` |

### Watchdog 监控系统

后台持续监控资源，超过阈值时输出告警。

```bash
agent-unix watch --duration 60    # 监控 60 秒（默认）
agent-unix watch --duration 10    # 监控 10 秒（演示）
```

**默认告警规则**：

| 指标 | Warning 阈值 | Critical 阈值 |
|------|-------------|---------------|
| 磁盘使用率 | > 80% | > 95% |
| 内存使用率 | > 85% | > 95% |

输出示例：
```
🔍 启动 Watchdog 监控系统
   监控时长：60 秒

⚠️ [WARNING] 磁盘 / 使用率已达 85.2%，超过阈值 80.0%
   时间: 14:30:15

🚨 [CRITICAL] 内存使用率已达 96.5%，超过阈值 95.0%
   时间: 14:30:45

✅ 监控已完成
```

## 安全机制

所有工具调用在执行前都经过五级风险评估：

| 级别 | 说明 | 行为 |
|------|------|------|
| **Safe** | 只读操作（查询、列表） | 直接执行 |
| **Low** | 影响有限的写操作 | 直接执行 |
| **Medium** | 中风险（创建文件、安装软件） | `--mode safe` 时需确认 |
| **High** | 高风险（停止服务、修改系统配置） | **必须输入 `yes` 确认** |
| **Critical** | 极危险（删除系统文件、`rm -rf /`） | **直接拒绝，不给确认机会** |

**Critical 拦截示例**：
```bash
agent-unix run "删除 /etc/passwd"
# 🚨 操作被强制阻止（CRITICAL）
# 原因：删除系统关键文件可能导致系统无法启动

agent-unix run "rm -rf /"
# 🚨 操作被强制阻止（CRITICAL）
```

**High 确认示例**：
```
⚠️ 高风险操作 — 需要您确认
   工具：shell.exec
   命令：sudo systemctl stop sshd
   风险：停止 SSH 服务将断开所有远程连接
输入 'yes' 确认执行，其他任意键取消 › yes
```

**运行模式控制风险确认范围**：

| 模式 | 说明 |
|------|------|
| `safe` | Medium 及以上都需确认 |
| `normal` | 只有 High 及以上需要确认（**默认**） |
| `auto` | 只拦截 Critical，其他自动执行 |

## 完整命令参考

### 命令列表

```
agent-unix <COMMAND> [OPTIONS]

命令：
  chat       进入交互式对话（默认 TUI）
  run        执行单条自然语言指令
  config     查看/配置 LLM Provider
  history    查看操作历史
  undo       撤销上一次操作
  playbooks  列出 Playbook 模板
  explain    反向解释配置文件
  watch      启动后台监控
  info       显示版本和系统信息
  help       显示帮助
```

### 全局参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--verbose` | 输出详细日志 | 关闭 |
| `--mode <MODE>` | 运行模式：`safe` / `normal` / `auto` | `normal` |
| `--provider <P>` | LLM Provider：`anthropic` / `openai` / `openrouter` / `groq` / `deepseek` / `custom` | 读取配置 |
| `--model <ID>` | 模型 ID | Provider 默认 |
| `--base-url <URL>` | 自定义 API 端点 | Provider 默认 |
| `--no-tui` | 禁用 TUI，使用传统 CLI | 关闭 |

## 常见问题

**Q1: 安装后执行 `agent-unix` 提示 command not found**

确认 `~/.local/bin` 已在 PATH 中：
```bash
echo $PATH
# 如果没有，执行：
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
```

**Q2: macOS 上 `systemctl: command not found`**

macOS 没有 systemctl，使用 `launchctl` 替代，或改用通用命令（`ps`、`top`、`df`）。建议在 Linux 上完整演示服务管理功能。

**Q3: 如何创建自定义 Playbook？**

```bash
mkdir -p ~/.agent-unix/playbooks
# 创建 JSON 文件，格式见上方"创建自定义 Playbook"章节
agent-unix playbooks  # 验证加载
```

**Q4: undo 能撤销哪些操作？**

| 操作 | 回滚方案 |
|------|----------|
| `apt install nginx` | `apt remove nginx` |
| `useradd testuser` | `userdel testuser` |
| `systemctl start sshd` | `systemctl stop sshd` |
| `rm file.txt` | 无法自动回滚（需备份） |
| `file.write` | 无法自动回滚（需备份原内容） |

**Q5: 如何使用 OpenRouter 访问多种模型？**

```bash
export AGENT_UNIX_LLM_API_KEY=sk-or-你的key
agent-unix --provider openrouter --model openai/gpt-4o chat
agent-unix --provider openrouter --model anthropic/claude-3.5-sonnet chat
```

**Q6: 如何完全重新编译？**

```bash
cd /path/to/agent-unix
cargo clean && cargo build --release
cp target/release/agent-unix ~/.local/bin/agent-unix
```

```
├── playbook/
│   └── manager.rs    # Playbook 多来源加载
├── explainer/        # 反向解释模式
├── watchdog/         # 后台监控系统
└── types/
    ├── tool.rs       # ToolCall / Playbook / RollbackPlan
    └── risk.rs       # RiskLevel 枚举
```
