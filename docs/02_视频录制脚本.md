# SysWeaver 演示视频录制脚本

> **建议时长：** 8–10 分钟
> **推荐工具：** OBS Studio 录屏 + 系统麦克风
> **终端字体：** JetBrains Mono 16pt，背景暗色，便于录制清晰
> **预先准备：** 导出 API Key、`cargo build --release` 完成、开启 TUI 模式

---

## 开场白（0:00 – 0:30）

**旁白（边说边开终端）：**

> "大家好，这是 SysWeaver —— 一个用 Rust 编写的操作系统智能代理。
> 它把自然语言变成可控、可审计的 Linux 系统操作，
> 所有执行路径都经过五级风险分类器把关，高危操作必须人工确认，
> Critical 级危险直接拒绝执行。
> 下面我们通过四个场景来演示它的核心能力。"

**操作：**
```bash
# 展示 release 二进制
ls -lh target/release/sysweaver
```

---

## Scene A：基础信息查询（0:30 – 2:00）

**旁白：**
> "首先是最基础的场景：让 SysWeaver 帮我查看当前系统状态。"

**操作：** 启动 TUI 模式
```bash
export ANTHROPIC_API_KEY=<your-key>
cargo run --release -- chat
```

**等 TUI 加载完成后，在输入框键入：**

1. 输入：`查看磁盘使用情况`
   - **旁白：** "注意 TUI 状态栏实时显示工具调用过程。Agent 自动选择了 system.info(query=disk)，经过 Safe 级别判定直接执行。"
   - 等待输出，镜头停留在结果上 3 秒

2. 输入：`列出内存占用最高的 5 个进程`
   - **旁白：** "Agent 先推理「我需要进程信息」，调用 process.manage，结果按内存排序后以表格呈现。"
   - 等待输出

3. 输入：`查看当前网络监听端口`
   - **旁白：** "net.check 工具扫描本机监听端口，结果包含协议、端口号、进程。"
   - 等待输出

---

## Scene B：高危操作拦截（2:00 – 4:30）

**旁白：**
> "现在演示最重要的能力：安全防护。我们依次测试三级拦截。"

### B-1：Critical 级 —— 直接拒绝（无确认机会）

**操作：** 在 TUI 输入框键入
```
rm -rf /
```

**旁白：**
> "这是一条会摧毁整个文件系统的命令。"

- 等待输出，TUI 显示红色告警 Modal / CLI 显示 [CRITICAL BLOCKED]
- **旁白：** "看，Agent 在安全层就被拦截了。不仅拒绝执行，还给出了：① 为什么危险 ② 安全替代方案 ③ 如何验证替代方案的效果。没有任何确认对话框——Critical 级不给用户说 yes 的机会。"

### B-2：Critical 级 —— 覆盖系统文件

**操作：**
```
删除 /etc/passwd
```

- 等待输出，观察拒绝信息
- **旁白：** "同样被 Critical 规则命中。/etc/passwd 是用户数据库，删除会导致系统无法登录。"

### B-3：High 级 —— 强制用户确认

**操作：**
```
停止 SSH 服务
```

- **旁白：** "停止 SSH 是 High 级别风险。TUI 弹出确认 Modal，展示「危险原因：将断开所有远程连接，包括当前 SSH 会话」+ 替代方案。"
- 在 Modal 中选择**取消（按 ESC 或 N）**
- **旁白：** "我选择取消。Agent 不会重试，转而给出安全替代方案。"

---

## Scene C：环境感知（4:30 – 5:45）

**旁白：**
> "SysWeaver 在启动时会扫描当前系统环境，自动感知包管理器和操作系统发行版。"

**操作：**
```
帮我安装一个 Web 服务器
```

- **旁白：** "我没有指定任何包名或命令。Agent 读取了系统上下文：包管理器 = apt（Ubuntu）或 yum（CentOS/openEuler），自动选择了对应的安装命令。"
- 等待 Agent 输出安装计划（dry-run 预览）
- **旁白：** "它先用 dry_run=true 预览了执行效果，再等待确认。这体现了「先预览、再执行」的行为准则。"

**（可选）** 切到 explain 命令演示环境感知
```bash
# Ctrl+C 退出 TUI，然后：
cargo run --release -- explain /etc/nginx/nginx.conf
```
- **旁白：** "explain 命令可以读取任意配置文件，让 LLM 用中文解释它的内容和作用。"

---

## Scene D：多步骤连续任务（5:45 – 8:00）

**旁白：**
> "最后演示多步骤任务。一句话，Agent 自动分解为有序步骤，每步完成后汇报状态。"

**操作：** 重新进入 TUI（或在之前 TUI 中继续）
```bash
cargo run --release -- chat
```

**输入：**
```
把 nginx 配置改到 8080 端口，然后重启服务，最后验证端口是否监听
```

- **旁白：** "这是一个需要 4 步的任务。观察 Agent 的执行计划输出。"

**等待 Agent 输出执行计划，逐步解说：**
- **旁白（Step 1）：** "第一步：file.read 读取 nginx.conf，Safe 级，直接执行。"
- **旁白（Step 2）：** "第二步：file.write 修改端口配置。它先给出 dry_run 预览——显示将要修改的行，然后 Medium 级确认（或 safe 模式询问）。"
- **旁白（Step 3）：** "第三步：service.manage restart nginx。Medium 级，继续执行。"
- **旁白（Step 4）：** "第四步：net.check 验证 8080 是否已在监听。自动验收！"

**等待"任务完成报告"输出**
- **旁白：** "最后 Agent 输出了结构化任务报告：每步状态（✅/❌）+ 总体结果 + 后续建议。"

**演示 Undo：**
```
/undo
```
- **旁白：** "输入 /undo，Agent 从回滚链中取出：恢复配置文件备份 + 重启 nginx。完整回滚，无需手动操作。"

## 结尾（8:00 – 9:00）

**旁白：**
> "总结一下 SysWeaver 的核心能力：
> 一、双协议 LLM 接入——Anthropic 原生 tool_use + OpenAI-compatible，国内外模型都能用；
> 二、五级风险分类——150+ 正则规则，Critical 直拒，High 强确认；
> 三、13 个域工具——覆盖用户、服务、进程、包管理、网络、定时任务等常见运维场景；
> 四、自动回滚——每个写操作都生成反向操作，/undo 一键还原；
> 五、环境自适应——自动识别 Ubuntu/CentOS/openEuler/macOS 并切换命令；
> 六、JSONL 审计日志——所有操作可追溯、可审计。
>
> 完整代码和文档见提交包，感谢观看！"

**最后操作：**
```bash
# 展示审计日志
cat ~/.sysweaver/audit.jsonl | tail -20 | python3 -m json.tool
```
- **旁白：** "这是今天操作的审计日志，每条记录包含时间戳、会话 ID、工具名称、风险级别和执行结果。"

## 补充建议

### 录制前检查清单
- [ ] `cargo build --release` 无 warning 无 error
- [ ] 设置好 `ANTHROPIC_API_KEY`（或其他 provider）
- [ ] 终端字体 >= 14pt，对比度足够
- [ ] 关闭桌面通知（录制期间不被打断）
- [ ] `~/.sysweaver/audit.jsonl` 清空一下，演示日志干净

### 可选加分场景（有时间可录）
| 场景 | 命令 | 说明 |
|------|------|------|
| Playbook 保存 | `/playbook save nginx-8080` | 把刚才多步任务保存为可复用剧本 |
| Playbook 运行 | `sysweaver playbook run nginx-8080` | 一键复现，无需再输入自然语言 |
| Watchdog 告警 | `sysweaver watch` | 后台监控磁盘/内存，超阈值告警推入 TUI |
| SSH 远程模式 | `sysweaver --ssh user@host chat` | 所有工具透明转发到远端执行 |
| dry-run 全程 | `sysweaver run --dry-run "清理 30 天前的日志"` | 展示预览而不真实执行 |

### 时间压缩版（5 分钟极速版）
如果时间紧张，可以只录 **B + D**（高危拦截 + 多步任务），其余通过文档说明。
Scene B 展示安全能力（最有说服力），Scene D 展示智能分解（最体现"代理"价值）。
