## 快速运行

```bash
# 1. 安装 Rust（如未安装）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 2. 构建
cd SysWeaver
cargo build --release

# 3. 配置 API Key（任选其一）
export ANTHROPIC_API_KEY=sk-ant-...
# 或 OpenAI-compatible：
# export SYSWEAVER_LLM_PROVIDER=openai-compatible
# export SYSWEAVER_LLM_BASE_URL=https://api.openai.com
# export SYSWEAVER_LLM_API_KEY=sk-...

# 4. 启动
./target/release/sysweaver chat          # TUI 模式
./target/release/sysweaver run "查看磁盘使用情况"   # 单条指令
./target/release/sysweaver run --dry-run "清理日志" # 预览模式
```

## Agent 形态说明（4.2.2）

### Agent 配置

| 参数 | 说明 | 来源 |
|------|------|------|
| Provider | `anthropic` 或 `openai-compatible` | 环境变量 / CLI |
| Model | `claude-sonnet-4-5`（默认）或任意兼容模型 | 环境变量 / CLI |
| Base URL | Anthropic 官方或自定义代理 | 环境变量 / CLI |
| Mode | `normal`（默认）/ `safe`（Medium 也需确认） | CLI `--mode` |
| Max Steps | 20 步（防死循环） | 硬编码 |
| LLM 重试 | 3 次 + 指数退避 | 硬编码 |

### 核心 Prompt 位置

| Prompt | 文件 | 作用 |
|--------|------|------|
| System Prompt | `src/llm/prompt.rs::build_system_prompt()` | 注入环境、规则、工具优先级 |
| Planner Prompt | `src/llm/prompt.rs::build_planner_prompt()` | 任务三分类（single/multi/ambiguous） |

### 工具定义

全部 13 个工具的 JSON Schema 在运行时通过 `ToolManager::schemas()` 动态生成，发给 LLM。详见 `src/tools/mod.rs` 及各工具的 `schema()` 方法。

主要工具：

| 工具名 | 能力 | 风险等级 |
|--------|------|---------|
| `system.info` | 查询 OS/CPU/内存/磁盘/网络/服务 | Safe |
| `shell.exec` | 执行 shell 命令 | Safe–Critical（按内容） |
| `file.read` / `file.write` / `file.search` | 文件操作 | Safe / Medium |
| `process.manage` | 进程列表/终止 | Safe / High |
| `service.manage` | 服务启停 | Safe–High |
| `user.manage` | 用户增删改查 | Safe–High |
| `package.manage` | apt/yum/brew 包管理 | Medium |
| `cron.manage` | 定时任务 | Low–Medium |
| `log.tail` | 实时日志 | Safe |
| `net.check` | 端口/DNS 诊断 | Safe |
| `health.check` | 磁盘/内存/服务健康 | Safe |

## 决策逻辑与执行路径（4.2.2 书面说明）

```
用户自然语言输入
    │
    ▼
[Planner] LLM 分类：single / multi / ambiguous
    │ single/multi                │ ambiguous
    │                             ▼
    │                       TUI 消歧 Modal
    │                       用户选 A/B/C
    │                             │
    ▼                             ▼
[Agent Loop] ReAct（最多 20 步）
    │
    ▼
LLM 推理 → 输出 ToolCall（结构化 JSON）
    │
    ▼
[RiskClassifier] 五级评估
    ├── Critical → 直接拒绝 + 审计日志 + 解释
    ├── High     → TUI Modal / CLI inquire 确认
    │              用户 yes → 继续 / 取消 → 停止
    ├── Medium   → safe 模式需确认，normal 直接执行
    └── Low/Safe → 直接执行
    │
    ▼
[ToolManager] 路由到对应工具
    │
    ├── Local subprocess（默认）
    └── SSH remote（--ssh 模式）
    │
    ▼
ToolResult → 回传 LLM 上下文 → 继续推理
    │
    ▼（stop 条件：LLM 输出最终文本 / 达到 20 步）
输出最终回复 + 写审计日志 + 更新 Memory 回滚链
```

## 操作覆盖范围

### 已覆盖（完整实现）
- 系统信息查询（磁盘/内存/CPU/进程/用户/服务/网络）
- 用户账户管理（新增/删除/修改密码/列表）
- 服务生命周期（启动/停止/重启/启用/禁用/状态）
- 进程管理（列表/搜索/终止/信号）
- 软件包管理（安装/卸载/更新/搜索，自适应 apt/yum/dnf/brew）
- 文件操作（读/写含自动备份/递归搜索）
- 计划任务（列表/添加/删除/编辑 crontab）
- 日志查看（实时 tail，支持过滤）
- 网络诊断（端口扫描/DNS 解析/连接状态）
- 系统健康巡检（磁盘压力/内存告警/服务异常）
- 多步骤任务（自动分解 + 顺序执行 + 失败恢复）
- 操作回滚（/undo 一键还原）
- Playbook（保存/加载/复用多步任务）
- 配置反向解释（explain 命令）
- SSH 远程执行（透明代理所有工具）
- 后台 Watchdog（磁盘/内存超阈值告警）

### 交互限制
- 不支持需要 stdin 交互的命令（如 `passwd` 原生交互）— 通过 user.manage 提供等价结构化接口
- 不支持 GUI 应用操作
- 超长命令输出（> 4000 字符）会被截断
